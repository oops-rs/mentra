# mentra v0.9.0

mentra evolves from a fixed loop into a **pluggable agent core**: additive,
generic, per-run extension points that let a host inject round policy, typed
terminal actions, opaque metadata, ephemeral persistence, and honest soft
budgets â€” without mentra learning any host concept and without changing the
behavior of any existing embedder. This release implements
[ADR-0001](../adr/0001-pluggable-agent-core.md).

## Highlights

### Per-run round strategy

- `RunOptions::round_strategy: Option<Arc<dyn RoundStrategy>>` â€” an async
  strategy owned by exactly one `Agent::run`, invoked after a committed tool
  round and after a committed tool-free assistant message before the run
  returns.
- Decisions: continue, inject committed corrective context, switch the next
  round's model/reasoning, or request a graceful stop (transcript committed,
  identical to `RunOptions::stop`). A stop never asserts answer validity.
- `None` is byte-identical to v0.8.0 behavior; strategy state cannot outlive
  its run or cross a pooled runtime.

### Structured tool output with termination

- `ToolOutput { content: ToolResultContent, details: Option<serde_json::Value>, terminate: bool }`
  beside the unchanged `ToolResult`; defaulted
  `ToolExecutor::execute_output`/`execute_mut_output` bridge existing string
  tools with zero source changes.
- `terminate: true` ends the run as the tool's own value. Terminal-marked
  descriptors (`.terminal()`) are never scheduled in a parallel batch; a
  parallel-lane terminate is rejected as misuse; calls scheduled after a
  termination receive explicit not-executed results â€” never a silent drop.

### Opaque transcript metadata

- `details` survives the local transcript and replay as a per-`tool_use_id`
  map on `TranscriptItem` (`with_details`/`details()`); provider requests
  receive only `content` through the single existing projection boundary.

### Volatile runtime profile

- `runtime::VolatileRuntimeStore` implements the full `RuntimeStore`
  composition in memory: an ephemeral run leaves no durable trace (agent/run
  rows, transcript upserts, leases, team/task writes, memory ingest,
  transcript files). Isolation on a retained store is an explicit seam
  (fresh construction per run, or `reset()`). The SQLite default is
  unchanged.

### Metadata-preserving compaction

- Documented, regression-locked guarantee: `details` on preserved and
  salvaged items survives `StandardCompactionEngine::compact` bit-for-bit,
  and the pre-compaction snapshot carries every item's details.

### Honest soft budgets

- `RunOptions::token_budget` â€” a round-boundary **soft** token bound over
  reported usage: the crossing round completes, the transcript stays
  committed, the run stops gracefully. Never claimed as a hard cap; no
  monetary surface exists.
- `RunOptions::child()` shares cancellation, stop, deadline, and token
  accounting with child runs; `RoundContext::transport_retries()` reports
  transport retries distinctly from logical rounds. Existing
  `model_budget`/`model_requests` semantics are unchanged.

## Compatibility

- Every seam defaults to current behavior; builder and
  `..Default::default()` users compile unchanged.
- Exhaustive literal constructors/matchers: `RuntimeToolDescriptor` gains a
  public `terminal` field, `RunOptions` gains `token_budget`/`token_usage`,
  and `RuntimeHookEvent::ToolExecutionFinished` gains a `details` field.
- Persisted transcripts/agent memory from earlier versions deserialize
  unchanged.

## Crate Versions

- `mentra 0.9.0`
- `mentra-provider 0.3.1` remains unchanged and does not need to be
  republished.
